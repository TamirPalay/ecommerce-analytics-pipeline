# E-Commerce Analytics Pipeline

## Overview

This project simulates a real-world analytics workflow for an e-commerce marketplace.  
It transforms raw transactional data into structured, queryable datasets and produces actionable business insights using a full analytics pipeline.

The project demonstrates:

- Data cleaning and transformation using Python (Pandas)
- Relational schema design using PostgreSQL
- Advanced SQL analysis (CTEs, window functions, aggregations)
- Customer segmentation (RFM methodology)
- KPI development and business interpretation
- Dashboard reporting using Power BI
- Reproducible and well-documented analytics workflows

---

## Dataset

This project uses the **Olist Brazilian E-Commerce Public Dataset** from Kaggle:

https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce

After downloading and extracting the dataset, place the CSV files in:

Data/Raw

## Business Objectives

This analysis was designed to answer key marketplace questions:

- How does revenue evolve over time?
- What drives revenue concentration?
- What percentage of customers are repeat buyers?
- Which product categories generate the most revenue?
- How efficient is delivery performance?
- What customer segments drive the majority of revenue?

## Database Design

The database follows a simplified star schema.

### Dimension Tables
- dim_customers — customer location attributes
- dim_products — product attributes and translated categories

### Fact Tables
- fact_orders — order-level metrics (timestamp, status, revenue, delivery time)
- fact_order_items — item-level sales and freight details

Foreign keys and indexes were implemented to ensure relational integrity and query performance.

## Key Insights

### 1. Revenue concentration among high-value customers

The "Champions" segment (25,271 customers) generated over **6.5M in total revenue**, making it the largest revenue-driving group.

### 2. Significant churn risk

Over **18,000 customers classified as "At Risk"** have not purchased in over 500 days on average, yet historically generated **3.7M in revenue**.  
This highlights strong reactivation potential.

### 3. Extremely low repeat purchase behavior

Average customer frequency is approximately **1 order per customer** across segments.  
This indicates that the marketplace is primarily driven by one-time buyers rather than long-term retention.

Implication:
- High acquisition reliance
- Low customer lifetime expansion
- Opportunity for loyalty or retention programs

### 4. Revenue concentration in product categories

A small subset of product categories contributes a disproportionately large share of total revenue, suggesting opportunities for inventory optimization and targeted marketing.

### 5. Delivery performance variability

Average delivery time varies significantly by region, with measurable long-tail delays (P90 delivery time), indicating potential operational improvement areas.

## How to Run Locally

### 1. Clone Repository

git clone https://github.com/TamirPalay/ecommerce-analytics-pipeline.git  
cd ecommerce-analytics-pipeline

### 2. Create Virtual Environment

python -m venv venv  
venv\Scripts\activate  

### 3. Install Dependencies

pip install -r requirements.txt

### 4. Download Dataset

Download from Kaggle and place CSV files in:

Data/Raw

### 5. Run Data Cleaning

python Src/data_cleaning.py  

Processed files will be written to:

Data/Processed

### 6. Create PostgreSQL Database

Create a database named:

ecommerce_db

### 7. Create Tables

Run:

SQL/schema.sql

### 8. Load Data into PostgreSQL

python Src/load_to_db.py

### 9. Run Analytics Queries

Execute:

SQL/analysis_queries.sql

### 10. Build Dashboard

Connect Power BI to:

Server: localhost  
Database: ecommerce_db  

Build visuals based on the metrics above.
