# E-Commerce Analytics Pipeline

## Overview

This project simulates a production-style analytics workflow for an e-commerce marketplace.

It transforms raw transactional data into structured, queryable datasets and generates business-ready insights using a full analytics pipeline.

This project demonstrates both **Data Analyst** and **Analytics Engineer** capabilities:

- Data cleaning and transformation using Python (Pandas)
- Relational schema design in PostgreSQL
- Data modeling using a simplified star schema
- Advanced SQL analytics (CTEs, window functions, NTILE segmentation)
- Creation of reusable SQL views for reporting
- Data quality validation checks
- KPI development and business interpretation
- Dashboard reporting in Power BI
- Clear documentation and reproducibility

---

## Dataset

This project uses the **Olist Brazilian E-Commerce Public Dataset** from Kaggle:

https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce

## Database Design

The database follows a simplified star schema.

### Dimension Tables
- dim_customers — customer location attributes
- dim_products — product attributes and translated category names

### Fact Tables
- fact_orders — order-level data (timestamps, status, revenue, delivery time)
- fact_order_items — item-level sales & freight values

Foreign keys and indexes are implemented to ensure relational integrity and query performance.

---

## Analytics Views (Production-Style Layer)

To simulate a production reporting layer, reusable SQL views were created:

### rfm_segments_summary
Customer segmentation using RFM methodology.

Outputs:
- Segment name
- Customer count
- Average recency
- Average orders
- Average spend
- Total spend per segment

### vw_monthly_kpis
Monthly KPI aggregation view.

Includes:
- Orders
- Revenue
- Average Order Value
- Average delivery time

### vw_customer_pareto
Revenue concentration analysis using running totals to measure 80/20 distribution.

Includes:
- Revenue per customer
- Customer rank
- Cumulative revenue percentage

These views are consumed directly by Power BI.

## Key Insights

### 1. Revenue concentration among high-value customers

The "Champions" segment (25,271 customers) generated over **6.5M in revenue**, making it the largest revenue-driving group.

### 2. Significant churn risk

Over **18,000 customers classified as "At Risk"** have not purchased in over 500 days on average, yet historically generated **3.7M in revenue**.

This highlights strong reactivation potential.

### 3. Extremely low repeat purchase behavior

Average customer frequency is approximately **1 order per customer**.

This suggests:
- Heavy reliance on new customer acquisition
- Low retention
- Strong opportunity for loyalty programs

### 4. Revenue concentration pattern

Revenue accumulation follows a clear Pareto-style distribution, where a minority of customers generate a large portion of total revenue.

### 5. Delivery performance variability

Delivery times vary significantly by region, with measurable long-tail delays (P90 metric), suggesting operational improvement opportunities.

## How to Run Locally

### 1. Clone Repository

git clone https://github.com/TamirPalay/ecommerce-analytics-pipeline.git  
cd ecommerce-analytics-pipeline  

### 2. Create Virtual Environment

python -m venv venv  
venv\Scripts\activate  

### 3. Install Dependencies

pip install -r requirements.txt  

### 4. Download Dataset

Download from Kaggle and place CSV files in:

Data/Raw  

### 5. Run Data Cleaning

python Src/data_cleaning.py  

Outputs are written to:

Data/Processed  

### 6. Create PostgreSQL Database

Create a database named:

ecommerce_db  

### 7. Create Tables

Run:

SQL/schema.sql  

### 8. Load Data into PostgreSQL

python Src/load_to_db.py  

### 9. Run Analytics Queries & Create Views

Execute:

SQL/analysis_queries.sql  

Create analytics views:
- rfm_segments_summary
- vw_monthly_kpis
- vw_customer_pareto  

### 10. Build Dashboard in Power BI

Connect to:

Server: localhost  
Database: ecommerce_db  

Import tables and views and build report pages.
